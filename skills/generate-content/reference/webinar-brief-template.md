# Nexudus Webinar Brief — Template & Guidelines

A webinar brief is a **structured host guide**, not a full script. It gives the hosts a clear
run-of-show with bullet-point talking points so they can deliver the webinar naturally in their
own words. The training team explicitly does not write a script for hosts to read verbatim.

Real examples are in `examples/webinar-briefs/`. Load one before generating — it takes
precedence over this template.

## Writing guidelines

### Metadata table
- **Marketing Name** — the public-facing webinar title. Should be benefit-led and audience-focused.
  Pattern: `[Outcome]: [How feature achieves it]` or a short punchy phrase
  - ✅ "Our new AI Front Desk Assistant"
  - ✅ "Fractional Offices: Turning Underused Offices into Revenue"
  - ❌ "NexDelivery 2.0 Update Webinar"
- **Moderators** — omit the row entirely if there is no moderator
- **Hosts** — link names to email addresses

### Note for hosts
Always include the standard note verbatim — it sets the right expectation with hosts.

### Main content table

Use a two-column table throughout. Left column = `Topic/Section`, Right column = `Points to touch on`.

**Standard sections (always include):**
1. Introductions
2. Housekeeping (Duration, Q&A, Recording)
3. Benefits / Feature Overview
4. [Feature-specific sections — as many as needed]
5. Live Q&A
6. Wrap up

**Explain / Demonstrate pattern:**
- Use `Explain:` for context, concepts, and talking points the host should convey verbally
- Use `Demonstrate:` for specific on-screen steps or flows — number these when order matters
- Not every section needs both — some sections are explanation only
- Be specific in Demonstrate steps: name the exact screen, setting, or flow, not just "show the feature"

**Navigational callouts:**
- For a full-width contextual note between sections (e.g., "Navigate to the new portal"), merge both columns into a single row in italics

### Q&A section
- The "Live Q&A" row in the table is a placeholder only — no pre-populated questions needed
- If there are known tricky questions the hosts should be ready for, add them as sub-bullets in the row

### Key dates
Include when known. Use the standard milestones:
- Slides Due
- Practice Session 1
- Practice Session 2

For special cases (e.g., a translated/companion session), adapt milestone names accordingly.
Omit the row if dates are not yet confirmed.

---

## Tone

- **The brief is written for the training team and hosts** — it's an internal working document
- **Direct and practical** — each bullet should be a clear action or talking point
- **No full sentences needed** — bullet fragments are fine ("Mention user role permissions")
- **No marketing language** — the marketing name is catchy; the brief content is functional

- `Virtual Offices - Webinar Brief.docx`
